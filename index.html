Healthcare Capstone Project
LangGraphOllamaSQLMulti-AgentDuckDB

Architected an offline, multi-agent LangGraph system to enable clinicians to query OMOP CDM healthcare data using natural language safely.

View Case Study →

The Problem

Clinical experts often face significant delays when trying to validate medical hypotheses because they rely heavily on specialized data engineers to query and extract information from complex, highly-regulated healthcare databases. The objective was to bridge the gap between clinical expertise and data science by building an intelligent copilot that allows medical professionals to query complex datasets directly using natural language.

The Solution

Engineered a multi-agent AI workflow using LangGraph and DuckDB. The system uses ClinicalBERT and FAISS to ground medical terminology to actual database schemas, dynamically generating accurate SQL queries while enforcing strict Human-In-The-Loop (HITL) checkpoints. This created an auditable, transparent analytics assistant that empowers clinicians to rapidly explore health data securely.

The Multi-Agent Workflow

  • Intent & Routing: Classifies the analytical type (cohort counting, trends, etc.). An Intent Clarifier detects underspecified questions and asks the user for clarification before forcing a query.
  • Schema Grounding: Maps natural-language concepts to known tables, fields, and controlled vocabularies using FAISS and ClinicalBERT, preventing the AI from hallucinating invalid medical columns.
  • SQL Generation (DuckDB): A dedicated SQL agent receives dynamic schema pruning to generate explainable DuckDB SQL, drastically reducing prompt clutter and errors.
  • Human-In-The-Loop Checkpoints: Positions the assistant as a copilot. Intermediate artifacts (query plans, generated SQL) are stored in a Data Registry for human review and auditing before sensitive analytical steps.
Judicial Analytics
GPT PipelineLegalBERTHANPyTorch

Developed a deep learning pipeline utilizing a Hierarchical Attention Network and SwiGLU to predict defendant liability from unstructured legal documents.

View Case Study →

The Problem

Standard text analytics models fail on legal texts due to dense terminology, rigid structures, and the high risk of "data leakage"—where models artificially inflate performance by reading outcome clues hidden within the judgment. The objective was to architect a specialized NLP pipeline capable of accurately extracting, processing, and analyzing complex judicial rulings without relying on post-decision factual material.

The Solution

I developed a custom hierarchical architecture using PyTorch, LegalBERT, and XGBoost, implementing rigorous data leakage controls (such as temporal isolation and IRAC-inspired extraction) to strictly separate pre-decision facts from final judicial conclusions. This delivered a scalable, robust framework for data-driven legal analytics that prevents artificial outcome inflation.

Pipeline Design & Architecture

  • Leakage Controls: Implemented strict temporal isolation, separate counterclaim handling, and metric realignment to ensure models learn only from pre-outcome information.
  • IRAC-Inspired Extraction: Structured facts, issues, and rules separately before final conclusions, utilizing LLM-as-a-Judge QA to ensure high-fidelity JSON data extraction.
  • Hierarchical Attention Network (HAN): Built a custom HAN to hierarchically encode legal text, preserving the separation between fact-level events and case-level representation.
  • Case-Level Aggregation: Converted party-level predictions into lawsuit-level signals aligned with practical legal-risk decisions, evaluated against XGBoost and LegalBERT baselines.
Predicting Loan Defaults
PySparkMachine LearningPCA

Built high-recall ML models to detect potential defaulters using 2.9M LendingClub records, utilizing PySpark and handling class imbalance.

View Case Study →

The Problem

Financial institutions face significant risk from loan defaults. Accurately predicting potential defaulters allows for better risk management and resource allocation. However, historical loan datasets are often massive, highly imbalanced (most people do not default), and feature-rich, requiring sophisticated preprocessing and modeling.

The Solution

I built and evaluated several machine learning models prioritizing a high recall rate to detect potential defaulters using LendingClub's extensive dataset spanning from 2007 to 2020 (comprising 2.9 million loan records).

Key Details & Methodologies

  • Big Data Processing: Leveraged PySpark to handle and process the 2.9 million records efficiently.
  • Feature Engineering & PCA: Cleaned the data and reduced dimensionality from 83 features down to 62 key components using Principal Component Analysis (PCA).
  • Handling Class Imbalance: Implemented SMOTE (Synthetic Minority Over-sampling Technique) and ENN (Edited Nearest Neighbours) to balance the target classes for better model training.
  • Model Evaluation: Trained and compared 6 different models including Logistic Regression, Random Forest, SVM, and Gradient Boosted Trees. Random Forest was selected as the best performer, achieving a recall of 0.8261 and a PR_AUC of 0.1890.
